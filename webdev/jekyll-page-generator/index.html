Static Page Generators

Easy Blogs and Static Websites with Jekyll

My first foray into Jekyll. I was looking for a way to create my own blog without the constraints of WordPress. Jekyll is a powerful, blog-aware static web site generator, and fits my needs.

So after some experimentation with Foundation 6, I made a basic website. With that, I could add articles to a page, but I was looking to for my site to do more, and do it automatically. I wanted to be able to track my blog articles with categories and tags, and have pages for each category and tag generated automatically, similar to what can be done in WordPress. I also wanted to limit the number of blog posts per page, and have a system for paginating my posts automatically as well. Having such automation would let me focus on the writing articles instead of managing the blog. At first, it seemed like I’d have to incorporate some JavaScript to have these pages generated.

After searching for an automated way of generating tag pages, I ended up finding Jekyll. It’s a blog-aware, static site generator. The concept is simple: design your page layouts and write your blog entries in simple Markdown format. It allows you to define partials, like the header and footer for your page layouts to reference. It’s a well structured framework to build a maintainable website. Just worry about the content, and not managing the web pages.

Blogging with Markdown is a simple and easy way to write compared to the markup syntax of HTML. No need to mind the beginning and end tags, and there’s a simpler syntax for things like headers and lists.

The system allows you to easily customize the layout and look of the site with its incorporation of the Liquid templating engine. With that, you can specify how the contents of your site displays on the page, using a mix of HTML and Liquid directives. There’s also Jekyll Bootstrap and Foundation templates available to make it easier to get a blog up and running. I was excited with Jekyll’s potential to be a very powerful tool for generating my site.

Jekyll’s installation process is all command line driven. That’s no problem for me, with my experience in UNIX shell scripts and commands. The installation process took longer than expected due to the older version of Ruby that comes with OS X Mountain Lion. After some failed attempts at installing a new version of Ruby, I decided to take the plunge and upgrade to OS X El Capitan. From what I read, my 2008 aluminum MacBook would be able to handle the upgrade, since I increased the RAM on my MacBook to 8GB a couple years ago. With some trepidation, I upgraded, and was pleasantly surprised with the process! The upgrade went smoothly, and I was up and running quite quickly. I love the look of El Capitan, and how smooth it runs. I wish I upgraded sooner!

After the upgrade, the Jekyll install went fairly smooth from that point on.

I found a great Foundation-Jekyll template, phlow’s Feeling Responsive. The template has almost everything I was looking for in a site template: a variety of blog page templates, a responsive design, a nice landing page, and different header styles. The documentation is pretty good, and easy to understand.

So, with that, I dove into the code for phlow, started to get a feel for how Jekyll worked, and started customizing the template.
