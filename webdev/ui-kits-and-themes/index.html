Web UI Kits and Themes

Bootstrap vs Foundation

I wanted to get a professional look to my blog without reinventing the wheel and writing my own UI framework. My search narrowed to the two most popular frameworks, Bootstrap and Foundation.

The framework of Bourbon, Neat, Bitters, and Refills makes it easy to get a simple static site up and running. Just copy and paste the things you need from the Bourbon site, and voilà, you have a website with some snazzy UI components. You choose the components you want to include, and you don’t have to include things you don’t need in your source code. The appeal here is that the framework is very lightweight and compact.

Installation of the Bourbon framework is done through command line utilities. While it’s not as easy as installing through a UI-based installer, it’s not difficult either. There’s a good demo video by DevTips which offers a good intro to Bourbon.

After playing around with it, I was able to get a skeleton of a website up. However, I wanted more from the framework. The concept is good, but it feels very bare bones.

While I like the minimalist concept of adding only what you need, it just seemed to be too much work customizing the look of the Bitters components. I prefer start off with a template with a website laid out and styled the way I want it to be. As much as I want to experiment with HTML and CSS, at this point, I just want to get a blog up and running quickly. I know I’ll have plenty of opportunity to customize and tinker with the website.

Because of that, I decided to continue my search for a UI framework better suited to what I wanted to do.

Bootstrap vs Foundation

After some reading and research, I found Bootstrap and Foundation, which are the most popular UI frameworks out there. Having choices is great, but there comes a point where I have to decide on which one to use. Vermillion gives a good comparison of the two frameworks here:

Both are very good and have similar features.

I ended up spending a lot of time browsing the large collections of free Bootstrap themes. There’s plenty of sites offering Bootstrap themes, with a great variety of color schemes, fonts, and landing pages.

Next, I looked at Foundation. Although there aren’t as many Foundation themes, this can be seen as an advantage as well, as too many similar choices can slow down the decision for which template to use. Instead of styled templates, with color and font selections chosen, Foundation provides a relatively unstyled set of templates

With the help of Yeti Launcher, it’s easy to create a website from scratch. It automates several tasks, such as compressing JavaScript with UglifyJS and removing unused CSS with UnCSS. My first goal is to get a site up and running, so I didn’t explore those features. However, I did use Yeti Launch to set up my website project, create a basic page layout template which incorporated the Handlebars templating language, and watch for changes in the SASS files for recompilation.

With that, I was able to get a basic site up and running, with a hero page, responsive menu bar, and a separate blog page complete with a sidebar.

Also, Bootstrap 3.x still uses px for sizing units. Bootstrap 4.0 switches to rems and ems, but it’s not a final version, and still in alpha. I was curious as to what those sizing units really mean. I found some very informative articles on responsive typography and rems vs ems

I decided to build my initial blog prototype with Foundation, because of Yeti Launch, with its automation features and Handlebars templating.
